## Running multiple node instances on the same codebase

You can run multiple node instances on the same codebase by using different server and db ports and different identity files.

You can create multiple script to separate the node instances using a run command like this:

`node1`

```sh
./run -u http://127.0.0.1:53550 -d 5332 -p 53552 -c true -i .demos_identity
```

`node2`

```sh
# different ports
./run -u http://127.0.0.1:53559 -d 5339 -p 53559 -c true -i .demos_identity2
```

> !TIP
> `demos_identity` is generated by default when you run the node for the first time.

You can uncomment lines 53 - 68 and modify line 53 in `src/libs/identity/identity.ts` to generate a new identity file.

Then add the public key of the identity file to `demos_peerlist.json` file and map it with their node url:

`demos_peerlist.json`

```jsonc
{
    "ccc...47a": "http://127.0.0.1:53550", // node1
    "a34...cd61": "http://127.0.0.1:53559", // node2
    "912...ccbe": "http://127.0.0.1:53560" // node3
}
```

Make the script files executable:

```sh
chmod +x node1
chmod +x node2
```

Finally, open different terminal windows and run the scripts:

Terminal 1

```sh
./node1
```

Terminal 2

```sh
./node2
```
